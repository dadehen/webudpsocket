UDP服务端需要公网IP，其作用是为该系统进行内网穿透

主机服务端是网络出口，可以与UDP服务端部署在同一主机，能直接部署在web云虚拟主机。目前将要开发的服务端支持的语言有C#,java,php

客户端是用户是对客户提供网络请求。目前将要开发的有windows，mac,android,ios

本系统能做很多事，比如网络加速，p2p，等。请不要利用本系统搭建翻墙系统


通信协议

通信格式：
表头+数据+校验
名称	格式	长度	备注
表头	byte数组	13	
数据	byte数组	X	
校验	byte数组	16	MD5校验表头+数据

表头格式：
名称	格式	长度	备注
类型	byte	1	U、S、P、D、G、T、R、C
用户	int	4	用户不能重复
用户编码	int	4	同一用户下的用户编码不能重复
长度	int	4	数据长度，数据长度


数据格式
U=>UDP服务器与网络进口和网络出口发送的心跳包
其数据为空	

S=>分两种情况
网络进口向UDP服务端发送为请求网络出口对其建立连接
名称	格式	长度	备注
用户	int	4	网络出口在UDP服务上的用户编号
用户编码	int	4	网络出口在UDP服务上的用户编码
UDP服务端向网络出口发送的穿透心跳指令，收到指令的机器向指定的IP端口名称	格式	长度	备注
IP数据	byte数组	4	
端口	int	4	

发送P格式数据：
P=>客户端和出口主机之间的相互心跳包
其数据为空

发送G格式数据：
G=> 获取数据
其数据为字符串，有 GetProxyServer、GetUserNo等。

发送D格式数据：
D=> UDP向网络进口和网络出口返回的数据
其数据为字符串 

发送T格式数据：
T和R是socket通信的控制字符串
其数据格式为字符串，有Connect Send   Close

发送C格式数据：
是socket通信的传输数据
名称	格式	长度	备注
Socket	int	4	
TCP编号	int	4	
UDP编号	int	4
长度	int	4
数据	byte数组	X
校验	byte数组	16	MD5校验该条传输
